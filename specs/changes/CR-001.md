# CR-001: D-skip mark mechanism

## Status: spec
## Sprint: 5
## Priority: P0
## Lane: backend（主） + pipeline（migration） + 少量 skill 改動
## Created: 2026-05-07
## Author: spec-writer (orchestrated)

---

## 1. 背景

v0.4.0 release 後實機運作發現 chat-drafts skill 行為缺陷：

- Skill 對 D 類訊息（純 ack「好」「OK」「收到」/ 別人之間對話 / 政策紅線命中）只在 stdout 印一行 `skipped: <reason>`，**不對 backend 做任何 mutation**。
- Backend `GET /api/claude/pending` 用「無對應 draft 紀錄」過濾 pending 訊息，所以這些被 skill skip 過的訊息**下一輪 query 仍出現**。
- `/loop` 每分鐘跑同一批，每輪重新對相同訊息判斷一次 → 完整 LLM call 成本（categorize round），純空轉。

**實測數據（2026-05-06 19:00–19:05）**：
- T0：26 則 pending，全部 D skip
- T1（+60s）：30 則 pending（含 4 則新 banter），全部 D skip
- T2（+120s）：32 則 pending，仍全部 D skip
- 5 分鐘內 categorize step 跑了 ~150 次相同決策

**目標**：讓 D skip 變成 backend 持久狀態，後續 pending list 自動排除，loop 真正 incremental。

---

## 2. 變更概要

### 2.1 新增

- **DB**：`messages.skipped_at TIMESTAMPTZ NULL`、`messages.skip_reason TEXT`、`messages.skipped_by TEXT`（migration 0018）
- **API**：
  - `POST /api/claude/skip` — skill / backend / manual 標記 skip
  - `GET /api/claude/skipped` — 列出最近 skip 的訊息（除錯 + 將來 UI）
  - `POST /api/claude/unskip` — 還原 skip（誤判修復）
- **Pending query 過濾**：`/api/claude/pending` SQL 增 `AND m.skipped_at IS NULL`
- **Backend 自動 skip**：blocked_keywords 命中、mention-only 模式下未被 mention 的訊息，由 backend 直接 skip（不用 skill 跑）
- **Skill 行為**：D 類判定後改為呼叫 `POST /api/claude/skip`，不再純 log
- **Backfill 工具**：`scripts/backfill-skip.sh`（或 `cmd/backfill-skip`），一次性把目前 pending 列表中 skill 已判過 D 但 backend 不知道的訊息標記 skipped

### 2.2 修改

- 受影響 features：
  - **F-002 Approval Queue**（一條新 scenario：被 skip 的訊息不出現在 inbox / approval queue）
  - **F-008 Safety rails**（不影響 spec，但測試環境 skipped 訊息要能正確被排除以避免雜訊）
  - **README / runbook**：說明 D-skip 機制 + backfill 流程

### 2.3 移除

無。

## 3. 候選方案決策

| 選項 | 結論 |
|------|------|
| A：messages.skipped_at + new endpoint | ✅ **採用** |
| B：沿用 /reply with status="skipped" | ❌ 拒絕 — 污染 drafts table，未來 sent log 會吃到雜訊 |
| C：Hybrid (A + bulk backfill 工具) | ✅ **同時採用** — A 為主結構，C 補一次性遷移工具 |

**為何選 A+C**：
- A 把 skip 與 draft 分離，語義乾淨：messages 表本來就是「已知訊息」的 source of truth，加狀態欄合理
- 不增加 drafts table 行數（drafts ≈ 「我們對外的回應」，skipped 訊息不該出現在這裡）
- C 解決現存資料卡住問題（不做 backfill，第一輪 skill 還是要跑一次每則才能 skip，浪費）
- 預留 audit 三欄（skipped_at / reason / by），日後做 UI undo / 統計都不用再 migrate

## 4. 詳細設計

### 4.1 Migration 0018（specs/contracts 新增）

```sql
-- internal/store/migrations/0018_messages_skipped.sql
ALTER TABLE messages
  ADD COLUMN skipped_at  TIMESTAMPTZ NULL,
  ADD COLUMN skip_reason TEXT NULL,
  ADD COLUMN skipped_by  TEXT NULL CHECK (
    skipped_by IN ('skill', 'backend_auto', 'manual', 'backfill')
  );

CREATE INDEX idx_messages_pending_active
  ON messages (created_at DESC)
  WHERE skipped_at IS NULL;

COMMENT ON COLUMN messages.skipped_at IS
  'Set when this message is intentionally not going to receive a draft. Filtered out of /api/claude/pending.';
COMMENT ON COLUMN messages.skip_reason IS
  'Free-form reason. e.g. "pure-ack", "blocked-keyword:money", "not-mentioned", "not-targeted"';
COMMENT ON COLUMN messages.skipped_by IS
  'Who marked: skill (Claude classified) / backend_auto (filter) / manual (UI) / backfill (one-time)';
```

### 4.2 API Contract

#### `POST /api/claude/skip`

**用途**：標記訊息為「不需 draft」。

**Request**：
```json
{
  "message_id": "msg_abc123",
  "reason": "pure-ack",
  "by": "skill"
}
```

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| message_id | string | yes | 必須存在於 messages 表 |
| reason | string | yes | 1..200 chars |
| by | string | no | enum: skill / backend_auto / manual / backfill；default `skill` |

**Response 200**：
```json
{
  "message_id": "msg_abc123",
  "skipped_at": "2026-05-07T03:14:00Z",
  "skip_reason": "pure-ack",
  "skipped_by": "skill"
}
```

**Response 200 (idempotent)**：若該 message 已被 skip，回現有狀態，不更新（不更新 skipped_at），HTTP 200。

**Errors**：
| Status | Code | Condition |
|--------|------|-----------|
| 400 | INVALID_INPUT | reason 為空或超過 200 字 / by 不在 enum |
| 404 | NOT_FOUND | message_id 不存在 |

#### `GET /api/claude/skipped`

**Query params**：
- `limit` (int, default 50, max 200)
- `since` (ISO 8601, optional) — 只回 skipped_at >= since
- `by` (string, optional) — 過濾 skipped_by

**Response 200**：
```json
{
  "items": [
    {
      "message_id": "msg_abc123",
      "space_key": "spaces/AAA",
      "sender_name": "Alice",
      "text": "好",
      "skipped_at": "2026-05-07T03:14:00Z",
      "skip_reason": "pure-ack",
      "skipped_by": "skill"
    }
  ],
  "next_since": "2026-05-07T03:14:00Z"
}
```

#### `POST /api/claude/unskip`

**Request**：
```json
{ "message_id": "msg_abc123" }
```

**Response 200**：清空三欄回 NULL，回該 message 目前狀態。

**Errors**：404 NOT_FOUND。

### 4.3 Pending query 變更

`internal/httpapi/claude.go` `pendingHandler`：

```sql
-- 既有 query 加一條
... WHERE
  m.skipped_at IS NULL
  AND NOT EXISTS (SELECT 1 FROM drafts d WHERE d.message_id = m.id)
  ...
```

### 4.4 Backend 自動 skip

於 chat_processor 收到新訊息、寫入 messages 後：

1. **mention-only 模式 + 未被 mention** → 立即 skip，reason=`"not-mentioned"`, by=`"backend_auto"`
2. **blocked_keywords 命中** → 立即 skip，reason=`"blocked-keyword:<keyword>"`, by=`"backend_auto"`
3. **sender 是 self（自己送的訊息）** → 立即 skip，reason=`"self-sent"`, by=`"backend_auto"`

→ 這幾類根本不送進 skill，省 token。

### 4.5 Skill 行為調整

`.claude/skills/chat-drafts/SKILL.md`：

當 categorize step 判定 D 類（pure-ack / overheard / policy-redline）：
- 改為呼叫 `POST http://localhost:8080/api/claude/skip` with `{message_id, reason: <D 子類>, by: "skill"}`
- 不再只 stdout print

D 子類 reason 字串 enum（給 skill 用）：
- `pure-ack` — 純確認回覆
- `overheard` — 別人之間對話
- `policy-redline` — 政策紅線（金錢承諾 / 敏感人物等）
- `not-targeted` — 訊息不指向 self
- `low-info` — 資訊量過低（emoji-only 等）

### 4.6 Backfill 工具

`scripts/backfill-skip.sh`（或 Go cmd）：

```
用法：
  ./backfill-skip --dry-run            # 列出將被 skip 的 message
  ./backfill-skip --apply              # 真的 skip
  ./backfill-skip --apply --max=50     # 限制數量

策略：
  1. SELECT m.* FROM messages m
     WHERE m.skipped_at IS NULL
       AND NOT EXISTS (SELECT 1 FROM drafts WHERE message_id = m.id)
       AND created_at < NOW() - INTERVAL '10 minutes'  -- 只 backfill 久未處理的
  2. 對每筆呼叫 skill categorize（或本地 heuristic：長度 ≤ 4 字 / 純 emoji / 命中 blocked_keywords）
  3. 命中 D 類 → POST /api/claude/skip with by="backfill"
  4. 結束印統計
```

僅在 sprint 5 release 時跑一次清舊資料，之後不需常駐。

### 5.3 回歸風險

| 風險 | 對策 |
|------|------|
| Pending query 加 filter 後漏算 | Migration 同時加 partial index `idx_messages_pending_active`；backfill 後跑 explain analyze 確認用 index |

| Drafts 表既有資料 query 不變 | 沒動 drafts table |
