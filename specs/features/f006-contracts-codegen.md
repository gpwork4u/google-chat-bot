# F-006: Contracts Codegen（Go struct → TS types 自動產生）

## Status: completed
## Sprint: 3
## Priority: P0
## Lane: pipeline

## 使用者故事

As a 開發者
I want Go DTO struct 自動轉成 TypeScript interface
So that 前後端型別同步，消除 PascalCase / snake_case 漂移，並在 CI 偵測 drift

## 範圍（In Scope）

1. 引入 [tygo](https://github.com/gzuidhof/tygo) 作為 Go → TS codegen 工具
2. 新增 `tygo.yaml` 設定，指定需要轉換的 Go package 和 output 路徑
3. 產出 `web/src/contracts.generated.ts`（頂端加 `// Code generated by tygo. DO NOT EDIT.`）
4. 新增 `Makefile` target `make contracts`（或 `npm run gen:contracts`）
5. 新增 CI workflow `.github/workflows/contracts-drift.yml`：PR 時偵測 Go struct 改變但未重產 → CI fail
6. 涵蓋核心 DTO：`Settings` / `Draft` / `Space` / `SentRecord` / `ProfileFact` / `Inbox`

## 非範圍（Out of Scope）

- 非核心 DTO 的轉換
- 前端元件改用 generated types（F-005 範疇）
- 變更 Go struct 定義

## 產出

- `web/src/contracts.generated.ts`（DO NOT EDIT 標頭）
- `tygo.yaml`（或同等設定）
- `Makefile` target `make contracts` 或 `npm run gen:contracts`
- `.github/workflows/contracts-drift.yml`

## 型別轉換規則

| Go | TypeScript |
|----|-----------|
| `*T` | `T \| null` |
| `time.Time` | `string` |
| 無 json tag 的欄位 | 不 export |

## 驗收標準

- `make contracts` 本機可執行並產出檔案
- CI workflow 在 PR 跑：刻意改 Go struct 不重產 → CI fail
- `web/src/contracts.ts` 從 `.generated` re-export 後 build 通過
- 產出檔頂端有 `// Code generated by tygo. DO NOT EDIT.`
- `*T` 對應 `T | null`，`time.Time` 對應 `string`，無 json tag 的欄位不 export
