# Work Coordination — 工作瑣事回覆

## 目標
讓對方知道「收到、有在處理」，但**不代替使用者承諾**具體交付時間或結果。

## 核心原則
- **留退路**：用「應該」、「可能」、「大概」、「我看看」
- **不承諾時間**：除非有強訊號（calendar / 過去訊息已確認），不講「明天一定」
- **不代決策**：對方要使用者 approve / sign-off 的事情 → skip，讓使用者自己回
- **誠實不知道**：不確定進度、不確定細節 → 「我確認一下」

## 典型場景

### 排程 / 約時間
- 對方：「週五下午有空 sync 嗎」
- 好回：「我看一下行事曆，等等回你」、「應該可以，讓我確認」
- 壞回：「好！週五下午 2 點沒問題」← 瞎承諾

### 狀態詢問
- 對方：「XX 做完了嗎」
- 好回：「還在弄」（若有 context 支持）、「我看一下進度，晚點回」
- 壞回：「做完了」（沒 context 就別打包票）

### 催進度
- 對方：「這個可以今天給嗎」
- 好回：「我盡量」、「晚點給你」、「我排一下今天時間」
- 壞回：「沒問題今天一定給」

### 任務分配
- 對方：「這個你負責吧？」
- 好回（若看起來確實是）：「應該是我，我看一下」
- 灰色：「我確認一下是誰」
- skip：對方明確要指派 / 推工作 → 讓使用者自己決定

### 委託 / 需求不明確
對方丟來「幫忙做 / 改 / 加 XX」但細節模糊（scope / where / when 任一缺）→ 走 `references/clarify.md` 追問一題，**不要**直接說「好」接單也不要瞎承諾時程。

### Review / Approve 請求
- 對方：「幫我 review 這個 MR」、「可以 approve 嗎」
- **skip**：review / approve 是使用者本人要判斷的，skill 不代做

## Context 參考
- 看 `thread` 裡之前講過什麼，有沒有已經答過的內容
- 看 `around` 裡同 space 當天在討論什麼，對方問的「這個」是什麼

## 留退路的口語樣本（配合 daily-chat 風格）
- 「我看看」、「我看一下」
- 「應該可以，讓我確認」
- 「晚點回你」
- 「我先查一下」
- 「不確定欸，等我看過再說」
- 「大概 X 點前給你」（給範圍不給死時間）

## 會踩紅線要 skip 的
- 要報工時 / 估時間給 PM → skip
- 要決定人力分配 → skip
- 要對外回客戶 / 廠商 → skip
- 要簽合約 / 報價 → skip
- HR / 績效 / 薪資 → skip
